Diferencias entre los tipos de Códigos

Es el conjunto de los programas de cómputo, procedimientos, reglas, documentación y datos asociados, que forman parte de las operaciones de un sistema de computación.

1. Código Fuente

Es el conjunto de instrucciones escritas en un lenguaje de programación de alto nivel (como Python, Java, C++) que los humanos pueden leer y entender fácilmente.

Características:

Contiene la lógica del programa expresada en un lenguaje sintáctico y semántico entendible por los programadores. Necesita ser traducido para que la máquina pueda ejecutarlo.

Un ejemplo de estos pueden ser Python, Java o C++

2. Código Objeto

Es el resultado intermedio que se obtiene después de traducir el código fuente mediante un compilador. Está en un formato de bajo nivel (lenguaje máquina) que la computadora entiende parcialmente.

Características:

No es ejecutable por sí mismo, ya que carece de ciertas referencias, como librerías externas. Es específico para una arquitectura de hardware y sistema operativo.

3. Código Ejecutable

Es el archivo final que resulta después de enlazar el código objeto y las librerías necesarias, convirtiéndolo en un programa completamente funcional.

Características:

Es directamente entendible por el procesador y puede ejecutarse en un sistema operativo específico. Normalmente tiene extensiones como .exe (Windows) o sin extensión específica (Linux/Unix).

Tipos de Traductores Los traductores son herramientas que convierten el código fuente a un formato ejecutable o interpretable por la máquina. Los principales tipos son:

a) Compiladores:

Función: Convierte el código fuente completo en código objeto o ejecutable en un solo paso.

Ventajas:

Genera programas rápidos y optimizados.

No requiere la presencia del compilador para ejecutar el programa.

b) Intérpretes

Función: Traduce y ejecuta el código fuente línea por línea, sin generar un archivo intermedio.

Ventajas:

Facilita la depuración porque permite observar errores en tiempo real.

Ideal para desarrollo y pruebas.

Ejemplo: Python, Ruby.

c) Ensambladores Función: Traduce programas escritos en lenguaje ensamblador (cercano al lenguaje máquina) a código objeto. Ventajas: Permite un control muy preciso del hardware. Ejemplo: NASM, MASM.
d) Enlazadores Función: Combina múltiples archivos de código objeto y librerías para generar el código ejecutable final. Ventajas: Resuelve referencias externas para crear un programa funcional. Ejemplo: ld (Linker de GNU). Con estos conceptos claros, puedes entender mejor cómo se transforma un programa desde su creación hasta su ejecución.

Relación entre ellos:

Código fuente: Escrito por el programador (alto nivel).

Código objeto: Compilado pero no ejecutable directamente.

Código ejecutable: Resultado final que puede ser corrido por el sistema operativo.

Relación entre ellos: Código fuente: Escrito por el programador (alto nivel). Código objeto: Compilado pero no ejecutable directamente. Código ejecutable: Resultado final que puede ser corrido por el sistema operativo.

La razón de ser de la ingeniería del software radica en abordar de manera estructurada y eficiente los retos asociados con el desarrollo , mantenimiento y gestión de sistemas de software. En esencia, busca garantizar que el software desarrollado sea:

  1. Calidad: Cumpla con los requerimientos funcionales y no funcionales establecidos.
  2. Confiable: Reduzca errores y falle con menor frecuencia.
  3. Mantenible: Pueda adaptarse a cambios futuros de manera eficiente.
  4. Escalable: Maneje el crecimiento en complejidad, usuarios o datos.
  5. Eficiente: Use recursos computacionales de forma adecuada.

El propósito principal es desarrollar soluciones de software que satisfagan las necesidades de los usuarios y organizaciones, optimizando recursos, reduciendo riesgos y asegurando la sostenibilidad del producto a largo plazo.

Razones Fundamentales:

  1. Complejidad Creciente del Software

    2. Los sistemas modernos son cada vez más complejos, interactivos y distribuidos. Sin una metodología formal, el desarrollo sería caótico e insostenible.

  2. Demanda de Alta Calidad

    3. El software afecta la vida cotidiana, desde aplicaciones bancarias hasta sistemas médicos. La calidad deficiente puede generar fallos graves.

  3. Eficiencia en Costos y Tiempo

    4. Un enfoque estructurado minimiza el retrabajo y asegura que los proyectos se entreguen dentro del presupuesto y plazo.

  4. Evolución Tecnológica

    5. El software necesita ser escalable y adaptable ante cambios tecnológicos o de requisitos.

  5. Trabajo en Equipo

    6. La ingeniería del software proporciona modelos y prácticas para coordinar equipos multidisciplinarios, asegurando cohesión y colaboración.

Metas de la Ingeniería del Software: Planificación y gestión eficaz de proyectos. Minimizar riesgos asociados con el desarrollo. Fomentar buenas prácticas mediante estándares. Optimizar el ciclo de vida del software, desde su concepción hasta su retiro. En resumen, la ingeniería del software existe para garantizar que el desarrollo de sistemas sea sistemático, y confiable, abordando los desafíos técnicos y organizativos del desarrollo de software moderno.

Ventajas de los lenguajes de marcas

  • Organización estructurada: Permiten representar datos en una jerarquía clara y comprensible, facilitando su análisis y manejo.
  • Interoperabilidad: Son estándares ampliamente aceptados, lo que garantiza la compatibilidad entre diferentes sistemas y plataformas.
  • Flexibilidad: Permiten personalizar la estructura de los datos para adaptarse a necesidades específicas.
  • Escalabilidad: Facilitan la ampliación o modificación de la estructura de datos sin afectar su integridad.
  • Separación de contenido y presentación: Ayudan a mantener los datos independientes de su forma de visualización, mejorando la reutilización.
  • Reutilización de datos: La misma información puede usarse en múltiples contextos o formatos sin duplicación innecesaria.
  • Facilidad de automatización: Su estructura clara permite un procesamiento eficiente por parte de las máquinas.
  • Estandarización: Al ser lenguajes basados en estándares, ofrecen consistencia y confianza en su uso.
  • Adaptabilidad a múltiples usos: Son útiles en una variedad de aplicaciones, como desarrollo web, intercambio de datos y almacenamiento estructurado.

1. Lenguajes de bajo nivel

Lenguaje máquina

Este lenguaje de marcas es el más antiguo que existe. El lenguaje máquina es el único que comprenden los ordenadores. Cualquier código redactado mediante lenguajes alternativos debe ser convertido a lenguaje máquina para que un ordenador pueda leerlo.

El lenguaje máquina se basa en el código binario, es decir, se escribe a base de 0 y 1. Aprender lenguaje máquina es algo muy complejo y no es útil para la creación de programas o webs. Es por esto que se han creado los demás lenguajes de bajo nivel, para poder redactar código de manera sencilla que puede ser convertido en lenguaje máquina a través de un ensamblador.

Lenguaje ensamblador

Los lenguajes de bajo nivel están muy relacionados con el lenguaje máquina. Ambos tipos de lenguaje tienen como objetivo ser comprendidos por un ordenador. La única diferencia es que los lenguajes ensambladores están hechos para ser escritos por humanos de manera sencilla. Pese a esto, tiene que ser traducido por un programa conocido como Ensamblador para que los ordenadores puedan comprenderlo.

En definitiva, el lenguajes ensamblador tiene las mismas funciones que el lenguajes máquina, pero está diseñado para poder ser escrito y comprendido con facilidad por los humanos. Los códigos son mucho más cortos e intuitivos y, además, tienden a ocupar mucho menos memoria.

2. Lenguajes de medio nivel

Los lenguajes de nivel medio tienden a ser clasificados como lenguajes de bajo nivel porque siguen dependiendo de ensambladores para poder ser comprendidos por los ordenadores. Pese a esto, permiten llevar a cabo tareas mucho más complejas que los lenguajes de bajo nivel como el uso de funciones. Aun así, muchas tareas deben ser llevadas a cabo de manera manual, aunque hay que destacar que permiten los algoritmos de búsqueda y ordenamiento a diferencia de otros lenguajes de alto nivel.

En ese sentido, los lenguajes de medio nivel están orientados hacia la programación de sistemas, por lo que no son adecuados para tareas como el diseño web. Los lenguajes de medio nivel más conocidos son C y C++.

3. Lenguajes de alto nivel

Los lenguajes de alto nivel son los más adaptados al lenguaje humano, por lo que son los más sencillos de aprender en términos generales. Como los de medio nivel, tienen que ser traducidos a lenguaje máquina para poder ser ejecutados por un ordenador. Pese a esto, debido a las modificaciones del lenguaje en relación al código máquina, los ensambladores no son útiles para esto. Los lenguajes de alto nivel se adaptan al código máquina a través de traductores y compiladores, dependiendo de si se requiere una ejecución más rápida o menos.

Actualmente, la gran mayoría de lenguajes de marcas existentes son de alto nivel, a excepción de C y C++. A continuación te dejamos los lenguajes de marcas más utilizados que existen y una breve explicación de sus usos:

HTML(HyperText Markup Language)

El lenguaje de marcas más conocido y utilizado es HTML, que es la base de la estructura de las páginas web. El lenguaje HTML utiliza una serie de etiquetas predefinidas para indicar elementos como títulos, párrafos, imágenes y enlaces, permitiendo a los desarrolladores crear documentos web estructurados y accesibles.

XML(eXtensible Markup Language)

El XML es un lenguaje de marcas que se utiliza para almacenar y transportar datos. A diferencia del HTML, que tiene un conjunto fijo de etiquetas, el XML permite a los usuarios definir sus propias etiquetas, lo que lo hace extremadamente flexible y adaptable a diferentes necesidades.

Este lenguaje se utiliza comúnmente en aplicaciones web, servicios web y sistemas de gestión de datos, ya que facilita el intercambio de información entre diferentes sistemas y plataformas.

Markdown

Otro lenguaje de marcas ampliamente utilizado es Markdown. Markdown es una herramienta de marcado ligera que se utiliza para formatear texto de una manera sencilla y legible. Fue creado para convertir texto plano en HTML de una forma intuitiva y fácil de leer, incluso en su forma sin formato.

Markdown se utiliza comúnmente en la redacción de documentación técnica, blogs y otros tipos de contenido en línea, ya que permite a los autores centrarse en la escritura sin preocuparse por el formato.

Necesidad de un lenguaje de marcas de propósito general

Los lenguajes de marcas de propósito general, como XML (Extensible Markup Language), son fundamentales para resolver problemas relacionados con la representación, intercambio y almacenamiento de datos en un formato que sea comprensible tanto para humanos como para máquinas. Su necesidad radica en las siguientes razones clave:

  • Interoperabilidad entre sistemas: Permite compartir datos entre sistemas heterogéneos sin perder información o estructura.
  • Estandarización y consistencia: Proporcionan un método uniforme para describir datos, reduciendo ambigüedades y errores.
  • Flexibilidad y extensibilidad: Permiten definir etiquetas personalizadas, adaptándose a necesidades específicas.
  • Separación de datos y presentación: Facilitan la reutilización de los datos al mantenerlos independientes de su formato de presentación.
  • Facilidad de integración: Pueden integrarse con otras tecnologías para facilitar el flujo de información.
  • Eficiencia en el tratamiento de grandes volúmenes de datos: Su estructura jerárquica permite organizar, filtrar y procesar datos de manera eficiente.

Ámbitos de aplicación específicos

Los lenguajes de marcas de propósito general tienen una amplia variedad de aplicaciones en diferentes sectores y contextos, incluyendo:

  • Intercambio de datos en sistemas distribuidos: Usados en APIs, servicios web y sistemas integrados para permitir la comunicación entre aplicaciones en diferentes lenguajes o plataformas. Por ejemplo, XML es común en protocolos como SOAP.
  • Gestión y almacenamiento de datos estructurados: Se utilizan en bases de datos orientadas a documentos para almacenar y recuperar información compleja, como archivos .xml.
  • Representación de documentos y metadatos: En bibliotecas digitales y sistemas de gestión de contenidos para etiquetar y organizar información textual, como artículos académicos o documentos legales.
  • Desarrollo de aplicaciones web y móviles: Permiten almacenar y procesar datos dinámicos que se consumen en aplicaciones cliente-servidor, como las configuraciones de interfaces en Android.
  • Comercio electrónico y finanzas: Se utilizan para estructurar y transmitir datos financieros, órdenes de compra, facturas electrónicas, etc. Por ejemplo, XBRL para contabilidad.
  • Interoperabilidad en sistemas de información sanitaria: Son esenciales para compartir registros médicos electrónicos y otros datos clínicos entre hospitales y organizaciones. Ejemplo: HL7 basado en XML.
  • Edición y publicación digital: Se emplean para estructurar el contenido que será procesado en múltiples formatos de salida, como PDF, HTML o ePub.

Clasificación de Lenguajes de Marcas Relacionados con la Web y sus Versiones

1. Lenguajes de marcas para estructura y contenido

  • HTML (HyperText Markup Language):
    • HTML 1.0 (1993): Proporciona una estructura básica para documentos web.
    • HTML 2.0 (1995): Introduce estándares básicos para interoperabilidad.
    • HTML 3.2 (1997): Añade soporte para tablas, scripts y diseño.
    • HTML 4.01 (1999): Mejoras en accesibilidad y formularios.
    • HTML5 (2014): Soporte para multimedia, gráficos y APIs avanzadas.
  • XHTML (eXtensible HyperText Markup Language):
    • XHTML 1.0 (2000): Basado en HTML 4.01 con reglas más estrictas.
    • XHTML 1.1 (2001): Mejoras para aplicaciones específicas.

2. Lenguajes de marcas para descripción de datos

  • XML (eXtensible Markup Language):
    • XML 1.0 (1998): Primera versión estándar.
    • XML 1.1 (2004): Compatibilidad con caracteres internacionales.
  • YAML (YAML Ain't Markup Language): Alternativa a XML y JSON para configuraciones y datos.

3. Lenguajes de marcas para presentación

  • CSS (Cascading Style Sheets):
    • CSS1 (1996): Primer estándar para estilos básicos.
    • CSS2 (1998): Diseño en capas y modelos de caja.
    • CSS3 (2011): Soporte para animaciones, transiciones y diseño responsivo.

4. Lenguajes de marcas para gráficos y multimedia

  • SVG (Scalable Vector Graphics):
    • SVG 1.0 (2001): Estándar para gráficos vectoriales escalables.
    • SVG 1.1 (2011): Compatibilidad mejorada con navegadores.
  • SMIL (Synchronized Multimedia Integration Language):
    • SMIL 1.0 (1998): Sincronización de audio, video y texto.
    • SMIL 2.0 (2005): Mejoras en funciones multimedia.

5. Lenguajes de marcas para comunicación

  • SOAP (Simple Object Access Protocol): SOAP 1.2 (2007): Basado en XML para servicios web.
  • WSDL (Web Services Description Language):
    • WSDL 1.1 (2001): Descripción básica de servicios web.
    • WSDL 2.0 (2007): Mejoras para servicios web modernos.

6. Lenguajes de marcas para semántica web

  • RDF (Resource Description Framework):
    • RDF 1.0 (1999): Representación de información sobre recursos web.
    • RDF 1.1 (2014): Mejoras en sintaxis y compatibilidad.

Funcionalidad de etiquetas y atributos en HTML

Etiquetas en HTML

Las etiquetas son los componentes fundamentales de HTML y se utilizan para definir la estructura y el contenido de una página web. Están compuestas por un nombre de etiqueta encerrado entre signos de menor que < y mayor que >.

Funcionalidad principal

  • Definir estructura: Organizan los elementos de la página en encabezados, párrafos, listas, tablas, etc.
    • Ejemplo: <h1>, <p>, <ul>.
  • Incorporar elementos multimedia: Permiten incluir imágenes, videos o audio.
    • Ejemplo: <img>, <video>.
  • Crear enlaces y navegación: Facilitan la conexión entre páginas web o secciones.
    • Ejemplo: <a>.
  • Mejorar la accesibilidad: Proveen una jerarquía y semántica clara que mejora la experiencia del usuario, incluyendo a personas con discapacidades.
    • Ejemplo: <nav>, <header>.

Atributos en HTML

Los atributos proporcionan información adicional o modifican el comportamiento de las etiquetas. Generalmente se colocan dentro de la etiqueta de apertura y consisten en un nombre y un valor.

Funcionalidad principal

  • Especificar propiedades de una etiqueta: Controlan detalles como tamaño, estilo o comportamiento.
    • Ejemplo: <img src="imagen.jpg" alt="Descripción">.
  • Definir identificadores únicos o clases: Facilitan el uso de estilos CSS o scripts JS.
    • Ejemplo: <div id="seccion1" class="contenido">.
  • Hacer enlaces dinámicos o interactividad: Con atributos como href, target o onclick.
    • Ejemplo: <a href="https://www.example.com" target="_blank">Visita</a>.
